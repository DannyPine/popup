# FloatingPortal

Portals the floating element into a given container element â€” by
default, outside of the app root and into the body.

```js
import {FloatingPortal} from '@floating-ui/react';

function Tooltip() {
  return (
    <FloatingPortal>
      {isOpen && <div>My portalled element</div>}
    </FloatingPortal>
  );
}
```

<Notice type="error" title="Important">
  Don't conditionally render the FloatingPortal, only its
  children as seen above.
</Notice>

## Props

```ts
interface Props {
  preserveTabOrder?: boolean;
}
```

### preserveTabOrder

default: `true{:js}`

When using non-modal focus management using
`<FloatingFocusManager />{:js}`, this will preserve the tab order
context based on the React tree instead of the DOM tree.

```js
<FloatingPortal preserveTabOrder={false}>
  {isOpen && <div>Tooltip</div>}
</FloatingPortal>
```

## FloatingPortalRoot

Using FloatingPortalRoot default portal container (body) can be
overridden by root prop.

```js
import {FloatingPortalRoot} from '@floating-ui/react';

function App() {
  return (
    <FloatingPortalRoot root="root">
      <Tooltip />
    </FloatingPortalRoot>
  );
}
```

## Props

```ts
interface Props {
  root: HTMLElement | null | string;
}
```

### root

You may pass the element or id as a string, By specifying a
string, if node does not exist on the document, it will be
created and appended to body.

## useFloatingPortalNode

Exposes the portal container node for custom use in other
components. The hook injects the node, if it does not yet exist
in the DOM.

```js
function App() {
  const portalNode = useFloatingPortalNode({
    id: 'custom-root-id',
    enabled: false, // true by default
  });

  if (portalNode) {
    return ReactDOM.createPortal(<div></div>, portalNode);
  }

  return null;
}
```
